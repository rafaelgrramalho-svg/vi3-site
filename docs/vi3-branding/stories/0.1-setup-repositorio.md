## Objetivo

Preparar o repositorio `vi3-institute` com scaffold Next.js 16, React 19, TypeScript, Tailwind CSS 4, e design tokens do Instituto VI3. Deploy de skeleton no Vercel para validar pipeline antes de qualquer desenvolvimento.

---

## Contexto

O repo `vi3-institute` ja existe no GitHub (`rafaelgrramalho-svg/vi3-institute`) com documentacao de branding em `docs/vi3-branding/`. Esta story inicializa o projeto Next.js dentro desse repo.

**Fontes de referencia:**
- `PRD-SITE-VI3.md` — Secao 5 (Stack) e 7 (Arquitetura)
- `tokens.json` — Design tokens DTCG
- `tokens-tailwind.js` — Mapeamento Tailwind v4
- `EPIC-SITE-VI3.md` — Scope 0.1

## Scope Tecnico

### 1. Scaffold Next.js
```bash
npx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir
```
- App Router (nao Pages Router)
- `src/` directory habilitado
- TypeScript strict mode (`"strict": true` no tsconfig)

### 2. Estrutura de Pastas (conforme PRD secao 7.2)
```
vi3-site/               # raiz do repo vi3-institute
├── src/
│   ├── app/            # 5 rotas SSG (futuro)
│   │   ├── layout.tsx  # Root layout (placeholder)
│   │   └── page.tsx    # Home (skeleton)
│   ├── components/
│   │   ├── atoms/
│   │   ├── molecules/
│   │   ├── organisms/
│   │   └── sections/
│   ├── lib/            # constants, metadata
│   └── styles/
│       └── globals.css
├── public/             # images, favicon (vazio por ora)
├── tokens.json
├── tokens-tailwind.js
├── tailwind.config.ts
├── tsconfig.json
└── package.json
```

### 3. Configuracao Tailwind 4
- Importar `tokens-tailwind.js` no `tailwind.config.ts`
- Garantir que classes de tokens funcionam (ex: `bg-navy`, `text-cyan`, `font-serif`)

### 4. globals.css
```css
@tailwind base;
@tailwind components;
@tailwind utilities;

:root {
  /* Cores primarias */
  --color-navy: #0A1628;
  --color-navy-mid: #0D1F3C;
  --color-navy-light: #122544;
  --color-cyan: #1E9FD4;
  --color-cyan-bright: #2EC4F0;
  --color-white: #F0F4FF;
  --color-white-dim: rgba(240, 244, 255, 0.65);
}

/* Focus visible global */
:focus-visible {
  outline: 2px solid var(--color-cyan);
  outline-offset: 2px;
}

/* Reduced motion */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

body {
  background-color: var(--color-navy);
  color: var(--color-white);
  font-family: 'DM Sans', sans-serif;
}
```

### 5. ESLint + TypeScript
- ESLint com config Next.js default
- TypeScript `strict: true`
- Rodar `npm run lint` e `npx tsc --noEmit` sem erros

### 6. Skeleton Page
- `page.tsx`: fundo navy (#0A1628), texto centralizado "VI3 — Em construcao" em branco
- Apenas para validar que pipeline funciona

### 7. Vercel Setup
- Criar Vercel project conectado ao repo GitHub
- Configurar dominio `vi3.com.br` (se acesso DNS disponivel)
- Primeiro deploy deve ser bem-sucedido

## Riscos

| Risco | Probabilidade | Impacto | Mitigacao |
|-------|---------------|---------|-----------|
| Tailwind 4 breaking changes vs formato tokens-tailwind.js | Media | Build falha | Validar compatibilidade do export format; ajustar config se necessario |
| Acesso DNS vi3.com.br indisponivel | Media | Deploy sem dominio customizado | Usar URL Vercel default; documentar instrucoes DNS para depois |
| Next.js 16 ainda em release candidate | Baixa | Bugs inesperados | Fixar versao especifica no package.json |
| Repo ja tem docs/ — scaffold pode conflitar | Baixa | Perda de arquivos | Usar create-next-app com cuidado, nao sobrescrever docs/vi3-branding/ |
